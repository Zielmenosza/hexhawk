Technical Preview Privacy

Preview privacy starts with local file handling and clear tester communication.

HexHawk is a local-first binary evidence workbench. During the technical preview, tester contact details and feedback are used to operate the small preview group, while file analysis remains local where applicable unless a feature explicitly says otherwise.

Local-first rule: Files are not uploaded unless a feature explicitly says otherwise. If a future feature sends data outside the machine, it should identify that behavior before use.
INFORMATION TESTERS SUBMIT

Application and feedback details.

Preview applicants and testers may choose to send information by email so HexHawk can evaluate fit, provide access, and understand bugs.

Application information

  • Name or preferred contact name.
  • Email address.
  • Organization or independent-tester context, if provided.
  • Windows version and basic test-system details.
  • Experience or intended workflows relevant to HexHawk testing.

Bug and usability reports

  • Package version, hash, install warning text, and reproduction steps.
  • Screenshots or logs you choose to send.
  • Descriptions of confusing labels, crashes, report-export issues, or expected-versus-actual behavior.
  • Any constraints you state about sensitive samples or disclosure.
LOCAL FILE HANDLING

Use non-critical files and systems.

HexHawk is intended to analyze files on your local test machine where applicable. You should still treat preview software cautiously and avoid critical machines, irreplaceable files, or sensitive production data.

Local analysisAnalysis is performed locally where applicable, and files are not uploaded unless a feature explicitly says otherwise.
Tester choiceDo not email samples, logs, screenshots, or paths that you are not comfortable sharing.
Preview riskUnsigned or early builds may trigger Windows or endpoint-security warnings and may have rough edges.
HOW INFORMATION IS USED

Operating the technical preview.

Submitted information may be used to contact applicants, decide whether the preview is a good fit, provide package-access instructions, debug reports, improve documentation, and prioritize product fixes. HexHawk should not be used for production-sensitive samples during this phase.
