HexHawk features

A reversing workbench with clear evidence boundaries.

IDA, Ghidra, and radare show the tool categories analysts expect: disassembly, decompilation, graphs, debugger or trace support, scripting, plugins, and documentation. HexHawk follows that familiar shape while making each result easier to review and keeping the final verdict source visible.

START WITH FACTS

Begin with things you can check.

Available

File identity

Input: an EXE or binary. Output: hashes, size, type, sections, imports, strings, packed/random-looking data clues, and architecture hints.

Available

Disassembly and graph clues

See instruction-level code views and control-flow clues so you can understand how the file is shaped.

Available

Readable-code guidance

TALON gives decompiler-style guidance when possible, with limits shown so it is not mistaken for perfect source code.

WORK FROM EVIDENCE

Keep clues together before sharing.

Internal tester

STRIKE trace notes

When configured, STRIKE can organize imported trace or debugger evidence. It does not replace human review and does not mean HexHawk is detonating files.

Available

Signature and pattern clues

ECHO compares exact and fuzzy signatures as clues. A match is evidence to review, not a magic answer.

Available

NEST evidence groups

NEST keeps related facts together so the report is easier to check later.

EXTEND AND AUTOMATE

Possibilities shaped by real analyst workflows.

Serious reversing work often needs repeatable scripts, plugins, command-line tools, and team-readable outputs. HexHawk is moving in that direction without claiming it already has every feature from long-established toolchains.

Internal tester

Plugin hooks

Pre-analysis, post-analysis, and custom resolver hooks let controlled plugins add evidence around the main analysis path.

Available

Headless and CLI surfaces

NEST CLI and headless STRIKE work support repeatable local checks where available.

Planned

Scriptable workflows

Future work should make repeatable analysis easier to run, record, and compare without hiding what was proven.

WHO DECIDES?

Engine names, translated.

Available

GYRE decides classification

GYRE is the only final verdict source. Its label and confidence must stay visible.

Helper

NEST organizes evidence

NEST helps collect and connect clues, like sorting puzzle pieces before a person checks the picture.

Optional

AETHERFRAME suggests improvements

AETHERFRAME can suggest safer wording or next steps when allowed. It must not secretly change the verdict.

OUTPUT

Reports another person can read.

Available

CREST reports

Package facts, notes, labels, and uncertainty into a handoff report.

Available

Source labels

Show which part produced each answer so reviewers know what they are looking at.

Important

Clear limits

If something was not proven, HexHawk should say so instead of pretending.
