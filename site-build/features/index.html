Capabilities

Understand what a binary is doing, step by step

HexHawk is built for working reverse engineers: open a binary, see the facts, follow the code, collect evidence, and export a report. Technical engines are still there, but the workflow starts with plain questions and visible results.

ANALYZE BINARIES

Start from facts you can check.

Available

File identity and metadata

Input: an EXE or binary. Output: hashes, sections, entropy, imports, strings, architecture hints, and risk clues.

Available

Disassembly and control flow

See instruction-level context and basic-block flow so you can move from suspicious strings to the code around them.

Improving

TALON decompiler guidance

Get readable pseudocode guidance with honest uncertainty. Recent SSA work adds safer fallback behavior for irreducible control flow instead of pretending every graph is source-like.
