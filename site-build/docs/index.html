Docs

Start here if you are new.

These notes explain HexHawk in plain language. If you know established reverse-engineering workbenches, think of HexHawk as a newer Windows workbench focused on evidence-first analysis and reviewable reports, not as a full replacement for every feature in those ecosystems.

First run

Use a test computer, open a safe sample file, and begin with hashes, imports, strings, sections, and code views.

Words to know

GYRE decides the final label. NEST organizes evidence. CREST makes reports. STRIKE handles trace evidence where configured. AETHERFRAME suggests improvements when allowed.

Limits

If a result was not proven, treat it as a clue, not a final fact. The Windows build is still unsigned.
