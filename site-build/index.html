Evidence-grade binary analysis

Reverse engineering with evidence you can review

HexHawk is a local-first desktop platform for malware analysts, incident responders, security researchers, and firmware teams. Inspect binaries, work through disassembly and decompiler output, converge evidence with NEST, preserve GYRE as final verdict authority, and export reviewable reports from one analyst workflow.

ANALYST WORKFLOW

One local workflow for binary evidence.

Analyze binaries in one local workflow

Inspect file identity, hashes, sections, entropy, imports, strings, and architecture-aware metadata without making cloud detonation the center of the workflow.

Review disassembly and decompiler output

Move between disassembly, control-flow context, and TALON decompiler assistance while preserving evidence context.

Converge evidence with NEST

Gather signals into reviewable bundles that can be validated, compared, and handed off to another analyst.

Preserve GYRE-linked verdict authority

GYRE remains the sole source of final classification and base confidence in HexHawk outputs.

Export CREST reports

Package findings for triage, review, diligence, and communication with explicit source and authority markers.

Keep refinement policy-gated

AETHERFRAME adds optional bounded metadata only when policy allows it. It does not replace GYRE or become final truth.

TRUST MODEL

Built for evidence, not opaque guesses.

HexHawk keeps authority boundaries explicit. GYRE decides. NEST organizes and validates evidence. AETHERFRAME adds optional policy-gated refinement metadata without changing classification. NEXUS assists the analyst without becoming the verdict source.
