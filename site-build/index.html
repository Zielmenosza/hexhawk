Reverse-engineering workbench

Open an unknown program. See facts, code views, evidence, and a report.

HexHawk is a Windows workbench for careful binary analysis. It is not trying to hide the hard parts behind a mystery score. It helps you inspect a file, move from facts into disassembly-style code views and readable-code guidance, connect evidence, and export notes another person can review.

WHAT PEOPLE EXPECT FROM A REVERSING TOOL

HexHawk covers the path from file facts to reviewable evidence.

Mature tools such as IDA, Ghidra, and radare show why analysts expect disassembly, decompilation, graph views, debugging or trace workflows, scripting, plugins, and clear documentation. HexHawk uses that same familiar map, but keeps each claim marked as available, internal tester, or planned.

Available

File facts first

Hashes, size, type, sections, strings, imports, entropy clues, and architecture hints before any final answer.

Available

Code views and graphs

Instruction-level views, control-flow clues, and readable-code guidance so reviewers can move from raw bytes to behavior.

Available

Evidence and reports

NEST groups clues, GYRE owns the final label, and CREST packages the result with uncertainty still visible.

Internal tester

Trace and debugger evidence

STRIKE is being shaped around imported trace and debugger evidence. It is evidence support, not automatic malware execution.

Internal tester

Plugin hooks

Pre-analysis, post-analysis, and resolver hooks are being added so controlled extensions can add evidence without taking over verdict authority.

Planned

Broader automation

CLI-friendly and script-friendly workflows are the direction, inspired by how serious reversing toolkits support repeatable analysis.

TRUST MODEL

Plain rule: facts first, people decide.

HexHawk labels where each answer came from. GYRE gives the final classification. NEST organizes evidence. AETHERFRAME can suggest safer wording or improvements when allowed. NEXUS helps the person. None of those helper parts should hide uncertainty.
