WHAT IT DOES

A simple workflow for a hard job.

1. Start with a file

Open an unknown EXE or binary and see the basics first: hashes, file type, sections, entropy, imports, strings, and architecture hints.

2. Read the code context

Move from strings and imports into disassembly, control-flow blocks, and TALON decompiler guidance. Recent GYRE SSA work now falls back safely on irreducible control flow instead of showing misleading variable versions.

3. Collect evidence

NEST organizes findings into reviewable evidence. STRIKE adds trace/debugger context when that workflow is configured and validated.

4. Keep the verdict clear

GYRE is the only final verdict source. Other layers can explain, organize, or refine metadata, but they do not silently change the classification.

5. Export a handoff report

CREST packages what was seen, what was inferred, what remains uncertain, and which engine supplied each part.

6. Know the limits

The current build is useful for controlled testing, demos, and diligence. It is not yet a signed public release or updater-ready distribution.

TRUST MODEL

Built for evidence, not opaque guesses.

HexHawk keeps authority boundaries explicit. GYRE decides. NEST organizes and validates evidence. AETHERFRAME adds optional policy-gated refinement metadata without changing classification. NEXUS assists the analyst without becoming the verdict source.
