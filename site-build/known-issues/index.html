Known Issues and Preview Limitations

Expected limits for an early HexHawk technical preview.

This page describes general preview limitations, not confirmed product defects. It is meant to help testers decide whether the preview is appropriate and what kinds of feedback are useful.

No production guarantee: Treat the preview as early software. Use non-critical Windows systems and non-critical files, verify package hashes, and keep independent analysis judgment.
EXPECTED PREVIEW LIMITATIONS

General limits testers should assume.

These are broad technical-preview cautions. They are not a list of confirmed bugs.

Unsigned builds

Windows warnings may appear

Packages may be unsigned or manually delivered. Windows SmartScreen, antivirus, or endpoint tools may warn, quarantine, or block installation or launch.

Manual channel

Updates may not be automatic

Preview builds may require manual delivery, hash verification, and reinstall steps. Do not assume public download, auto-update, or signed-release behavior.

Rough workflow

UX may change quickly

Labels, module names, report wording, and workflow order may be revised as testers identify confusing or incomplete areas.

Evidence limits

Output still needs review

HexHawk helps organize local evidence and labelled helper output. It should not be treated as a guarantee that an unknown file is safe or malicious.

File safety

Use non-critical samples

Test with files and systems you can afford to inspect in an early tool. Keep backups and do not rely on preview results as your only decision path.

Local handling clarity

Watch feature boundaries

Analysis is performed locally where applicable. Files are not uploaded unless a feature explicitly says otherwise; report any place where that boundary is unclear.

WHAT TO REPORT

Useful feedback for early testers.

Bug and usability reports should be specific enough for HexHawk to reproduce or understand the problem without requiring sensitive sample disclosure.

Install issuesPackage hash, Windows version, warning text, block/quarantine behavior, and exact step where it happened.
Workflow confusionWhat you tried to do, what label or page was confusing, and what wording would have helped.
Report outputMissing source labels, unclear uncertainty, export gaps, or places where helper output appears too authoritative.

Can you test within these limits?

If the preview constraints fit your setup, apply to join the small tester group. If you need signed, procurement-ready deployment, wait for a later channel.
