Technical Preview Application

Apply to test HexHawk before the broader release.

The preview is for a small group of technical testers who can safely run unsigned Windows software, inspect local files on non-critical systems, and send practical feedback about install, analysis, reporting, and usability problems.

Email your applicationReview preview limitationsRead privacy notes
Preview fit: Apply only if you can use controlled Windows test systems, non-critical samples, and clear bug-reporting discipline. HexHawk is not a signed public release yet.
WHO THIS IS FOR

Careful testers who understand early software risk.

HexHawk needs feedback from people who can exercise real binary-analysis workflows without treating preview output as a production verdict.

Good fit

Technical Windows testers

Reverse engineers, security analysts, technically curious operators, and early users who can describe what happened and what they expected.

Not a fit

Production-only environments

Do not apply if you need signed installers, enterprise deployment, procurement-ready support, or analysis on critical files and machines.

Feedback focus

Install and workflow clarity

Useful reports include warning dialogs, hash checks, confusing labels, unclear evidence paths, report export gaps, and steps to reproduce.

BASIC REQUIREMENTS

Windows and technical expectations.

Preview testers should be comfortable with a private, possibly unsigned package and with basic file-handling precautions.

WindowsA controlled Windows system suitable for testing unsigned or early-access software.
SafetyUse non-critical files, non-critical systems, and backups where needed. Do not use the preview as your only analysis path.
ReportingAbility to capture version, package hash, Windows warning text, reproduction steps, screenshots when safe, and expected versus actual behavior.
WHAT YOU MAY BE ASKED

Application information.

Applicants may be asked to provide only practical preview-selection and support details.

About you and your test setup

  • Name or preferred contact name.
  • Email address for preview communication.
  • Organization or independent-tester context, if relevant.
  • Windows version and whether you can use a non-critical test machine or VM.
  • General experience with binary analysis, malware triage, reverse engineering, or technical QA.

About your intended testing

  • What workflows you want to evaluate.
  • Whether you can report install warnings, crashes, confusing UX, and report-export issues.
  • Any constraints around sharing screenshots, sample names, or logs.
  • Confirmation that you understand builds may be unsigned and should be used only on non-critical systems.

How applicants will be contacted.
