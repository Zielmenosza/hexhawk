Trust & Assurance

What HexHawk decides, and what it only suggests

A reverse-engineering tool should not hide the difference between facts, guesses, and final verdicts. HexHawk labels those roles so an analyst can review the output instead of trusting a black box.

Simple rule: facts first, verdict second

HexHawk shows file facts and code evidence first. GYRE owns the final classification and base confidence. NEST organizes the supporting evidence. AETHERFRAME and NEXUS can help explain or package work, but they do not become the final security truth.

Authority

GYRE decides

Other engines can provide evidence, metadata, or assistance, but they do not replace GYRE classification.

Evidence convergence

NEST gathers, organizes, validates, and packages evidence linked to GYRE outputs. It improves reviewability without replacing verdict authority.

Optional refinement

AETHERFRAME adds bounded, replayable refinement and lineage metadata only when policy allows it. It does not change GYRE classification.

Analyst assistance

NEXUS helps the analyst navigate and act, but it does not become the security truth source.

High-assurance workflows

High-assurance use cases can disable uplift, preserve direct GYRE and NEST outputs, and require explicit disclosure when refinement metadata is present.
