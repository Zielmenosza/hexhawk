Trust & Assurance

Every answer should say where it came from.

Reverse-engineering tools can show many kinds of clues: bytes, strings, graphs, pseudocode, traces, scripts, and plugin output. HexHawk’s trust rule is that each clue should keep its source label, and the final verdict source must stay visible.

GYRE decides

GYRE gives the final classification and confidence. Helper tools must not secretly change that label.

NEST organizes

NEST groups evidence so reviewers can see which clues belong together.

AETHERFRAME suggests

AETHERFRAME may suggest safer wording or improvements, but only as a helper and only when allowed.

CHECKLIST

What a trustworthy result should show.

  • Which file was checked.
  • Which facts, code views, trace notes, or plugin outputs were used.
  • Which engine or helper produced each result.
  • Which parts are uncertain.
  • Whether the installer or report was signed or unsigned.
