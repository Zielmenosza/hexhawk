Trust & Assurance

Every result should say where it came from.

Reverse-engineering tools can show bytes, strings, graphs, pseudocode, traces, scripts, signatures, and AI-written notes. HexHawk’s trust rule is simple: keep the source visible and keep the final verdict source separate from helper evidence.

Authority

GYRE decides

GYRE gives the final classification and base confidence. Use the verdict as the answer to review, then inspect its supporting signals and contradictions.

Evidence

NEST organizes

NEST groups related facts and convergence notes around the GYRE-linked case. It helps reviewers see which clues belong together, but it does not become the final classifier.

Runtime notes

STRIKE imports evidence

STRIKE is for trace/debugger/runtime evidence that was collected through an approved workflow. It helps organize timelines and behavior notes. It should not be presented as HexHawk running unknown files for you.

Code guidance

TALON and ProgramAnalysis

TALON gives readable-code guidance. ProgramAnalysis structures functions, calls, references, imports, and graph context. Both support review; neither is recovered source or final classification.

Correlation

ECHO signatures

ECHO compares exact and fuzzy signatures as clues. A signature match should link back to evidence and remain reviewable.

Assistant

AETHERFRAME and NEXUS

AETHERFRAME can improve drafts or add bounded metadata. NEXUS can help the analyst navigate and write. Both are advisory and must stay labelled.

CHECKLIST

What a trustworthy result should show.

Evidence shown

  • Which file was checked.
  • Which facts, code views, trace notes, signatures, or plugin outputs were used.
  • Which engine or helper produced each result.
  • Which parts are uncertain or missing.

Limits shown

  • Whether runtime evidence was imported or not available.
  • Whether AI text was used and who approved it.
  • Whether an installer/report is signed or unsigned.
  • What a human should verify next.
