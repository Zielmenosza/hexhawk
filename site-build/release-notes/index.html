Release notes

2026-06-02 release-truth and website alignment update

This update keeps public release language aligned with the current unsigned internal-tester build candidate and documented updater/signing gates.

What changed in the latest release-truth pass

  • Current rebuilt Windows executable, MSI, and NSIS artifacts are unsigned according to Authenticode.
  • The prior internal self-signed evidence is historical and applies to older artifact hashes, not the current rebuilt artifacts.
  • The no-op Tauri signing command was removed.
  • Local unsigned builds keep updater artifacts disabled.
  • Updater key custody is present in GitHub Actions secrets and official-path metadata can be produced, but hosted metadata must match the current artifacts before endpoint-readiness claims.
  • Native packaged GUI report/AETHERFRAME policy parity is documented for the current unsigned MSI evidence.

Known caveats

  • Not a public-trusted general-availability release.
  • Expect Windows trust warnings until real code signing is configured and verified.
  • Rerun native GUI parity on the exact signed artifact intended for external testers or public release.
  • Do not treat stale hosted updater metadata or historical self-signed evidence as current release proof.
