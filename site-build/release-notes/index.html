RELEASE NOTES

2026-06-01 release-hardening, parity, and decompiler quality update.

This pass combines installer hardening work with TALON/decompiler reliability upgrades targeted at C-like output quality.

What changed

  • Enabled Tauri updater signing path in config (`createUpdaterArtifacts: true`, updater pubkey configured, endpoint wired).
  • Added release-hardening automation to sign exe/MSI/NSIS and emit signer/timestamp/hash evidence JSON.
  • Ran packaged native GUI acceptance probe on MSI-extracted artifact (runtime proof + Open/Inspect/Analysis/NEST/Report export).
  • Added report-export regression gate for authority envelope fields (`source_engine`, `gyre_is_sole_verdict_source`).
  • Aligned disassembly/CFG address handling to reduce non-overlapping function windows.
  • Added instruction-derived fallback block partitioning when CFG coverage is missing or non-overlapping.
  • Improved call argument recovery (larger lookback + cross-block recovery pass) and first-pass semantic variable naming heuristics.
  • Added new decompiler regressions, including guarded real-binary tests against workspace challenge executables.
  • Regenerated MSI and NSIS bundles from the updated source tree in this pass.

Known caveats

  • Signatures currently use an internal self-signed development certificate and are not publicly trusted.
  • Updater endpoint metadata validation currently fails because the configured release host did not resolve in this pass.
  • Tauri updater signing key (`TAURI_SIGNING_PRIVATE_KEY`) is not configured in this environment, so build exits non-zero after bundle generation.
  • Current internal packaging pass used a no-op sign-command fallback to unblock local MSI/NSIS rebundling.
  • Rerun packaged GUI parity on organization-trusted signed artifacts before external release.
