Public scope policy

  • This site publishes only public verification material: public keys, revocations, detached signatures, and checksums.
  • Private signing keys, certificate private material, and internal operational secrets are never published.
  • Automated tools should rely on machine-readable endpoints under /trust and /.well-known.

Verification workflow

1) Integrity check

Compare your file hash against published SHA-256 values from the downloads checksum ledger.

2) Signature check

Verify a detached signature in-browser using a published public key from the Keys and Revocations registry.

3) Revocation check

Confirm the signing key and release channel are not revoked before treating a build as trusted.

Recommended trust decisions

  • Internal evaluation: allow internal-signed artifacts with documented caveats.
  • External pilot: require sponsor acceptance of trust-chain and updater-endpoint constraints, or complete organization-trusted signing first.
  • Public distribution: require organization-trusted signing, endpoint health, and parity rerun on trusted-signed artifacts.
