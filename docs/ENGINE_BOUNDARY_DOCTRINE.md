# HexHawk Engine Boundary Doctrine

Date: 2026-05-31

This doctrine is mandatory for HexHawk code, documents, website copy, investor materials, and board reporting.

## Authority Hierarchy

1. GYRE is the sole verdict authority. It owns classification and base confidence.
2. NEST orchestrates evidence and convergence. It may select, package, and validate GYRE-linked outputs, but does not replace GYRE.
3. AETHERFRAME/Forge is optional bounded uplift/refinement/lineage. It may add confidence metadata only when policy permits; it must not mutate classification.
4. TALON, STRIKE, and ECHO are evidence/analysis surfaces.
5. CREST packages evidence and reports.
6. NEXUS is an assistant/consumer layer and is not security truth.

## Public Claim Rules

HexHawk copy must not claim that the product:

- detonates malware unless a specific controlled feature is implemented and validated;
- bypasses protections;
- proves exploitability, shell access, or flags;
- lets AI decide final security truth;
- guarantees malware classification accuracy.

Preferred language:

- evidence-grade reporting;
- local-first analysis;
- GYRE-linked verdicts;
- NEST evidence convergence;
- optional AETHERFRAME lineage/uplift;
- bounded, replayable, auditable refinement.
