## Factory Graduation Path

The Factory existed to bootstrap AetherFrame discipline: evidence-first intake, protected-boundary checks, narrow changes, validation, reports, lessons, and stop conditions.

The Factory should not become permanent bureaucracy. Once AetherFrame has native docs, templates, reporter behavior, checklists, and lessons, Factory framing becomes redundant.

Future cycles should be called **AetherFrame Advancement Cycles**. Factory files should be archived, not erased, unless the user explicitly approves removal after evidence preservation.

## Compatibility note

`scripts/aetherframe_factory_cycle.py` still exists as a backward-compatible read-only reporter. It should evolve into an AetherFrame Advancement reporter, but not through a risky rewrite in one pass.
