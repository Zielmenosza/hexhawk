# HexHawk Early Access Buyer Note

Channel: HexHawk Early Access — Unsigned Founder Build

## What you are buying

You are buying early private access to an unsigned HexHawk technical preview for Windows, plus the opportunity to influence the product through practical tester feedback.

The build is for technical testers who understand that unsigned Windows software may trigger operating-system or endpoint-security warnings.

## What you are not buying

You are not buying:

- a signed release;
- a Microsoft-verified application;
- a public/world-ready consumer release;
- an enterprise/procurement-ready package;
- an auto-updating application;
- guaranteed malware detection accuracy;
- a promise that Windows or your endpoint-security stack will trust the package automatically.

## Early-access limitations

- Artifacts are unsigned unless the exact package manifest says otherwise.
- Windows warnings are expected.
- Updates are manual.
- Support process and refund policy are still being formalized.
- Installer smoke, Function Notebook/export proof, updater readiness, and signing proof are separate evidence gates.

## Feedback request

Please report:

- install warnings or blocks;
- MSI vs NSIS behavior;
- startup failures;
- confusing UI wording;
- sample workflow issues;
- Function Notebook/export problems;
- incorrect or unclear advisory evidence;
- missing docs that would make testing safer.

Feedback improves HexHawk by guiding the factory process: docs, package gates, installer smoke tests, native workflow probes, support materials, and product hardening.

## Why paid unsigned early access exists

The first paid builds help fund:

- Authenticode code signing;
- updater signing and release metadata;
- exact-artifact release gates;
- installer and native workflow smoke automation;
- support and onboarding;
- product hardening before public release.

Payment/private distribution is not the same thing as signed/public release readiness. The channel stays honest by labeling the package unsigned and requiring hash verification.

## Authority boundary reminder

GYRE remains HexHawk's sole verdict/classification authority. Function Intelligence, AETHERFRAME, AI/NEXUS, TALON, STRIKE, and report packaging remain advisory or evidence surfaces and do not override GYRE.
